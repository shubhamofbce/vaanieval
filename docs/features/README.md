# Feature Playbooks For Copilot And Agents

This folder documents major product features in an implementation-first format for coding agents.

Goals:
- Help agents find the right entry points quickly
- Prevent cross-feature regressions
- Standardize change and validation checklists

## Folder Layout

- [Backend Feature Playbooks](backend/README.md)
- [Frontend Feature Playbooks](frontend/README.md)

## How To Use These Docs

1. Pick the feature page that matches your task.
2. Read Scope and Non-goals first.
3. Start from Entry Points.
4. Follow the Change Checklist and Validation Checklist.
5. Update this documentation if behavior or contracts change.

## Conventions

Each feature page includes:
- Scope
- Entry points
- Data/contracts
- Common agent tasks
- Validation checklist
- Known pitfalls
